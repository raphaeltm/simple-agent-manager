# Competitor Profile: Ona (formerly Gitpod)

**Last Updated**: 2026-03-11
**Category**: Direct
**Website**: https://ona.com (redirects from gitpod.io)

## Overview
Ona, formerly Gitpod, pivoted from cloud development environments to an AI software engineering platform. They position as "The AI software engineer you can rely on" — offering background agents that take a task and produce a pull request. Heavy enterprise focus with Fortune 500 customers.

## Target Market
- Fortune 500 enterprises (BNY Mellon, Blackstone, Vanta, Pearson)
- Teams doing large-scale code migration and modernization
- Organizations needing AI-powered code review at scale
- Enterprise security/compliance-conscious teams

## Product
- **Background Agents**: "Task in, pull request out" — autonomous end-to-end execution
- **Automations**: Agent fleets triggered across codebases via PRs, schedules, or webhooks
- **Connected Environments**: Full cloud environments with tools, network access, permissions
- **Governance**: VPC deployment, audit trails, scoped credentials, kernel-level policy enforcement
- Enterprise use cases: COBOL migration, Java modernization, CVE remediation across hundreds of repos

## Pricing
- "Start for free" tier available
- Enterprise pricing via demo/contact
- No public pricing details

## Strengths
- Brand recognition from Gitpod era + enterprise pivot
- Strong enterprise customer logos (Fortune 500)
- Claims: "4x productivity increase", "83% of PRs co-authored by Ona"
- SOC 2 certified, GDPR compliant
- Full autonomous agent capability (not just environments)
- VPC deployment for enterprise security

## Weaknesses
- Pivot from CDE to AI agent platform may confuse existing Gitpod users
- Enterprise-only focus — no clear path for individuals or small teams
- Pricing opacity
- Vendor lock-in concerns (managed platform, not BYOC)
- Less community/open-source presence post-pivot

## Recent Moves (as of 2026-03-11)
- Full rebrand from Gitpod to Ona
- Pivoted from CDEs to AI software engineering
- Targeting large-scale enterprise automation (code migration, CVE remediation)
- Dev Container spec support maintained

## Strategic Direction
All-in on enterprise AI software engineering. Competing with Devin/Cognition for the "autonomous AI engineer" space, but with enterprise governance angle.

## Key Differentiators vs SAM
- **Ona advantage**: Enterprise brand trust, Fortune 500 customers, autonomous agent fleet capability, VPC deployment, SOC 2
- **SAM advantage**: BYOC model, transparent/simple architecture, self-hostable on Cloudflare, open infrastructure, individual developer accessibility, not locked to their agent
