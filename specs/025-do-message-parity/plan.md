# Implementation Plan: DO Message Parity with Agent Stream

**Branch**: `do-message-parity-with-agent` | **Date**: 2026-03-08 | **Spec**: `specs/025-do-message-parity/spec.md`
**Input**: Feature specification from `/specs/025-do-message-parity/spec.md`

## Summary

The DO message persistence pipeline drops thinking blocks, plan items, and diff content, and creates duplicate tool call cards because `toolCallId` is not preserved. This plan enhances the VM agent's `ExtractMessages()` to capture these missing types, enriches `ToolContentItem` with actual diff content, preserves `toolCallId` for deduplication, and updates the frontend `chatMessagesToConversationItems()` to merge tool call updates and render thinking/plan items.

## Design Decisions

### D1: No DO Schema Migration Required

The `chat_messages.role` column is `TEXT NOT NULL` ‚Äî not an enum. Adding `role = "thinking"` and `role = "plan"` values requires no migration. The DO persistence code already handles arbitrary role strings.

### D2: ToolCallId Preservation Strategy

Add `ToolCallId` field to `ToolMeta` struct. Both `tool_call` and `tool_call_update` notifications include `toolCallId` from the ACP SDK. Store it in `ToolMeta.ToolCallId` so the frontend can deduplicate.

### D3: Tool Call Deduplication in Frontend

`chatMessagesToConversationItems()` will merge DO messages sharing the same `toolMetadata.toolCallId` into a single `ToolCallItem`. The last update's status, title, and content take precedence.

### D4: Diff Content Storage

`extractStructuredContent()` currently stores `c.Diff.Path` as the diff text. Enhanced to store `{type: "diff", path, oldText, newText}` as a JSON object in the `Text` field (or add new fields to `ToolContentItem`).

Given that `ToolContentItem` has `Type` and `Text` fields and the frontend already handles `type: "diff"`, the most backward-compatible approach is to expand `ToolContentItem` to include `OldText`, `NewText`, and `Path` fields for diffs. This matches the ACP SDK's `ToolCallContentDiff` structure.

### D5: Thinking Block Storage

Extract `agent_thought_chunk` as `role = "thinking"` with content = text. On the frontend, convert to `ThinkingItem` with `streaming: false` (these are persisted, complete blocks).

### D6: Plan Storage

Extract `plan` as `role = "plan"` with content = JSON-serialized plan entries. On the frontend, parse and convert to `PlanItem`.

### D7: Content Size Limits

Diff content (`OldText`, `NewText`) can be very large. Cap at `MAX_TOOL_CONTENT_SIZE` (env var, default 100KB) per diff field. Truncate with `"... [truncated]"` marker.

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Large diffs bloat DO storage | Medium | Medium | Content size cap (MAX_TOOL_CONTENT_SIZE) |
| Thinking blocks add noise to chat history | Low | Low | Render collapsed by default |
| Tool call dedup misses edge cases | Low | Medium | Fallback: orphan updates shown as standalone cards |
| Existing tests break | Low | High | Run full test suite before and after |
