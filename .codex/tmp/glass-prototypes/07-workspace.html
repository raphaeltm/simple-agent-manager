Found the root cause. Safari's Intelligent Tracking Prevention (ITP) blocks the session cookie because it's set during a cross-origin redirect from GitHub. The cookie uses SameSite=Lax, which Safari treats differently during the OAuth callback flow.

The fix involves two changes:

1. Set SameSite=None; Secure on the session cookie during the OAuth callback specifically

2. Add cookie partitioning support via Partitioned attribute for CHIPS compatibility
