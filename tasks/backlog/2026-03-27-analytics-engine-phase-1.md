# Analytics Engine Phase 1: Server-side Foundation

## Problem Statement

SAM currently has no analytics or usage tracking. We need visibility into user behavior, feature adoption, and conversion funnels. Cloudflare Analytics Engine is the natural choice — it's built into the Workers platform, requires no external services, and provides SQL-based querying.

This task implements the server-side analytics foundation: binding setup, request-level middleware that writes data points, event name mapping, admin API routes that query Analytics Engine, and an admin UI page for viewing the data.

## Research Findings

### Key Files
- `apps/api/wrangler.toml` — top-level config only, env sections generated by sync script
- `apps/api/src/index.ts` — Env interface (line 55-334), route registration (line 657 for admin)
- `apps/api/src/routes/admin.ts` — existing admin routes with `requireAuth() + requireApproved() + requireSuperadmin()` middleware
- `apps/api/src/middleware/auth.ts` — auth middleware patterns
- `apps/web/src/pages/Admin.tsx` — admin layout with `ADMIN_TABS` array and `<Tabs>` + `<Outlet>` pattern
- `apps/web/src/App.tsx` — route registration at line 91-98
- `apps/web/src/components/admin/HealthOverview.tsx` — admin component pattern (Card, Spinner, Button from UI lib)
- `apps/web/src/hooks/useAdminHealth.ts` — hook pattern for admin data fetching
- `scripts/deploy/sync-wrangler-config.ts` — extracts static bindings (DO, AI, migrations) but NOT analytics_engine_datasets

### Sync Script Gap
The `extractStaticBindings()` function only copies `durable_objects`, `ai`, and `migrations` from top-level to env sections. We need to add `analytics_engine_datasets` to the static extraction so it gets copied to generated env sections. Without this, the binding won't exist in staging/production.

### Env Interface Pattern
The `Env` interface at `apps/api/src/index.ts:55` lists all bindings. `AnalyticsEngineDataset` is a Cloudflare Workers type already available in the runtime.

### Admin Route Pattern
Admin routes use per-route or wildcard `use('/*', requireAuth(), requireApproved(), requireSuperadmin())`. Routes mounted at `app.route('/api/admin', adminRoutes)`.

### Admin UI Pattern
- `Admin.tsx` uses `ADMIN_TABS` array, `<Tabs>` component, `<Outlet>` for child routes
- Individual pages import components from `components/admin/`
- Hooks follow `useAdmin*` pattern with loading/error/refresh states
- Components use `Card`, `Spinner`, `Button`, `Body` from `@simple-agent-manager/ui`
- Tailwind CSS v4 classes throughout

### CF_ACCOUNT_ID
Already in Env interface at line 83 — available for Analytics Engine SQL API calls.

### CF_API_TOKEN
Already in Env interface at line 81 — available for Analytics Engine SQL API calls.
